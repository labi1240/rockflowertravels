# UI Context

## Theme

Dark technical workspace. No light mode[cite: 5]. The design language uses layered, near-black backgrounds, crisp surfaces, and high-vibrancy accent colors for critical interactive elements (like pricing charts or referral clicks)[cite: 5]. 

## Colors

All color custom properties are defined natively using HSL format rather than Hex to allow the AI to dynamically adjust lightness and saturation for accessible hierarchy[cite: 9]. Greys are saturated with a cool blue tint (`hsl(222, 15%, ...)`)[cite: 9]. Primary text and muted text maintain a strict 4.5:1 contrast ratio[cite: 9]. Very light or very dark utility shades increase saturation slightly to avoid looking washed out[cite: 9].

| Role | CSS Variable | HSL Value | Purpose / Description |
| :--- | :--- | :--- | :--- |
| Page background | `--bg-base` | `hsl(222, 15%, 6%)` | Main deep dark page background[cite: 5] |
| Surface base | `--bg-surface` | `hsl(222, 15%, 12%)` | Cards, search panels, and main container layers[cite: 5] |
| Surface elevated| `--bg-overlay` | `hsl(222, 15%, 18%)` | Dropdowns, tooltips, and floating modals |
| Primary text | `--text-primary` | `hsl(210, 20%, 98%)` | Headers, active values, high contrast data text[cite: 5] |
| Muted text | `--text-muted` | `hsl(215, 15%, 65%)` | Labels, secondary metadata, tertiary timestamps[cite: 5] |
| Primary accent | `--accent-primary` | `hsl(142, 70%, 45%)` | Vivid green for interactive actions, referral buttons, price indicators[cite: 5] |
| Subtle Border | `--border-default` | `hsl(222, 15%, 18%)` | Low-contrast edge boundaries (used sparingly)[cite: 5] |
| Error State | `--state-error` | `hsl(0, 84%, 60%)` | Destruction, delete steps, errors, critical price drops[cite: 5] |
| Success State | `--state-success` | `hsl(142, 76%, 40%)` | In-stock tags, verified elements, completed upgrades[cite: 5] |

## Typography

All text sizes use `rem` or `px` definitions[cite: 6]. The AI must never use `em` for font sizes to avoid unpredictable subpixel rounding errors on nested elements[cite: 9]. Mixed font sizes sharing a line must align strictly by their baseline, not vertical centers[cite: 9]. Headlines use decreased letter-spacing, and all-caps labels use increased letter-spacing for legibility[cite: 9].

| Role | Font | Variable | Style Guidelines |
| :--- | :--- | :--- | :--- |
| UI text | Geist Sans | `--font-sans` | Used for standard interface copy, forms, table data, and primary layout items[cite: 5] |
| Code/mono | Geist Mono | `--font-mono` | Used for precise product model values, data codes, dimensions, and numbers[cite: 5] |

## Border Radius

Consistency is mandatory[cite: 9]. Square and highly rounded corners must not be mixed within the same UI context[cite: 9].

| Context | Class | Purpose |
| :--- | :--- | :--- |
| Inline / small UI | `rounded-sm` (4px) | Badges, small checkboxes, and tight input buttons[cite: 5] |
| Cards / panels | `rounded-md` (8px) | Product cards, layout blocks, dialog content views[cite: 5] |
| Modals / overlays | `rounded-lg` (12px)| Global search overlays, center dropdown structures, dialog container frames[cite: 5] |

## Component Library

- Built using **shadcn/ui primitives** stacked natively on Tailwind 4.0[cite: 5]. All primitives live inside `components/ui/`[cite: 5]. 
- **Protected:** Primitives generated by the CLI are read-only[cite: 3]. The AI must compose components via wrapper layout extensions or Tailwind class combinations rather than refactoring the underlying library primitives[cite: 3, 5].

## Layout Patterns

- **Aggregator Dashboard Grid:** A full-viewport layout containing a narrow, fixed-width left filtering rail (`320px` max), and a flexible right grid executing automated data updates[cite: 5]. Sidebar elements use a fixed configuration rather than fluid percentages to prevent awkward text truncation on wide viewports[cite: 9].
- **Fewer Borders, More Depth:** Prioritize subtle background color changes (`--bg-base` vs `--bg-surface`) or layout gaps before inserting a border separator[cite: 9]. 
- **Two-Part Shadows:** Elevated elements (like product detail dialogs) use a two-part shadow configuration: a large, soft outer shadow mapping the virtual z-axis, paired with a tight, dark shadow underneath where ambient light is completely blocked[cite: 9].

## Icons

- **Library:** Lucide React (Stroke-based linear iconography exclusively)[cite: 5].
- **Sizing Balance:** Icons are inherently heavy in dark workspaces[cite: 9]. To maintain balance, they must be de-emphasized using softer colors (`--text-muted`) rather than matching primary text contrast[cite: 9].
- **Utility Constraints:**
  - Standard Inline/Text Icons: `h-4 w-4`[cite: 5]
  - Button/Action Indicators: `h-5 w-5`[cite: 5]
  - Large feature showcases: Wrap a standard `h-5` icon in a circular background instead of scaling the vector up to prevent chunkiness[cite: 9].
